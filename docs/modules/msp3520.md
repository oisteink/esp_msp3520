# 3.5" SPI TFT Module (MSP3520)

Source: https://www.lcdwiki.com/3.5inch_SPI_Module_ILI9488_SKU:MSP3520

## Specs

- **Display**: 3.5" TFT, 480x320, RGB 65K color
- **Driver IC**: ILI9488
- **Touch**: resistive, XPT2046 (separate SPI bus)
- **Interface**: SPI (display) + SPI (touch) -- two independent buses
- **Supply voltage**: 3.3V or 5V
- **Logic IO voltage**: 3.3V TTL
- **Power consumption**: 0.62W
- **Luminance**: 250 cd/m² (with touch) / 400 cd/m² (without touch)
- **Active area**: 48.96 x 73.44 mm
- **PCB size**: 56.34 x 98 mm
- **Weight**: 57g (with touch)
- **SD card slot**: yes (directly accessible on FPC/board; no header pins exposed)
- **Variants**: MSP3520 (with touch), MSP3521 (without touch)

## Pinout

14-pin header, active-low CS and RESET, active-high LED backlight.

### Display SPI

| Pin | Label | Function |
|-----|-------|----------|
| 1 | VCC | 3.3V/5V power input |
| 2 | GND | Ground |
| 3 | CS | LCD chip select (active low) |
| 4 | RESET | LCD reset (active low) |
| 5 | DC/RS | Data/command select (high = data, low = command) |
| 6 | SDI (MOSI) | SPI data in (MCU -> display) |
| 7 | SCK | SPI clock |
| 8 | LED | Backlight enable (high = on) |
| 9 | SDO (MISO) | SPI data out (display -> MCU) |

### Touch SPI

| Pin | Label | Function |
|-----|-------|----------|
| 10 | T_CLK | Touch SPI clock |
| 11 | T_CS | Touch chip select (active low) |
| 12 | T_DIN | Touch SPI data in (MCU -> touch) |
| 13 | T_DO | Touch SPI data out (touch -> MCU) |
| 14 | T_IRQ | Touch interrupt (low when touched) |

## Constraints

- **DC/RS polarity**: the lcdwiki page describes pin 5 as "high level: register, low level: data" but ILI9488 datasheet convention is DC high = data, low = command. Treat as standard DC pin.
- **LED pin** must be driven high (or connected to VCC) to enable the backlight; leaving it floating may result in a dark display.
- **Display and touch have separate SPI header pins** but can share a single SPI peripheral with separate CS lines. Confirmed working on SPI2 with ESP32-S3 (display at 40 MHz, touch at 1 MHz -- ESP-IDF SPI driver handles clock switching per-device).
- **SDO (MISO)** on the display side is needed only for reading back registers; many projects leave it unconnected and use write-only SPI.
- **3.3V logic**: despite accepting 5V power, all IO signals are 3.3V TTL. Safe to connect directly to ESP32 GPIOs.
- **No SD card header pins** are broken out on the 14-pin connector. The SD card slot is on the board but requires separate access.
